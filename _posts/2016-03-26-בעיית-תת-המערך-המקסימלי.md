**הערה:** כפי שהעיר הגולש akopel, יש טעות בהסבר, אך נראה שלא באלגוריתם המקורי שהוצע. בהסבר נפלתי בדיוק לטעות הקלאסית שהאלגוריתם נועד לפתור, ועצרתי את הסדרה מוקדם מדי, למרות שאפשר היה להמשיך אותה ולקבל פתרון מקסימלי יותר. אבל אחרי שהרצתי את הקוד כפי שהוא (ואספק דוגמת הרצה בהמשך), האלגוריתם לא עשה את הטעות שעשיתי בראש, והחזיר את הערך המקסימלי הנכון, 18, ולא 17. 
בעיה שחוזרת בכל דור ודור היא - (שנתקלתי בה ב Hackerrank תחת - Dynamic Programming) כזו:
יש לנו מערך בגודל מסויים, וברצוננו למצוא את הסכום המקסימלי שניתן להניב ע"י חיבור איברי תת מערך של המערך שקיבלנו.  מה זאת אומרת תת מערך?
נניח וקיבלנו את המערך <span dir="ltr" style="color:CadetBlue"> {2,3,-4,7,9,1}</span>
אזי, תת מערך לדוגמה הוא - <span dir="ltr" style="color:CadetBlue" >{-4,7,9}
 </span>

במקרה הזה, מהסתכלות קצרה במערך ~~קל לראות~~ (***תיקון:** אכן קל לראות, רק שזה לא נכון*) שתת המערך המקסימלי הוא - <span style="color:CadetBlue"> {2,3,-4,7,9,1} ~~{7,9,1}~~</span>. 
בשאלה המקורית מ-Hackerrank מבקשים גם למצוא את הסכום המקסימלי שניתן לקבל ע"י צירוף כלשהו של איברי המערך, לאו דווקא "רצופים". מה שמופיע פה בהמשך בתור Max_Comb_Sum.

![תת מערך מקסימלי](https://raw.githubusercontent.com/Tony-Stark/Tony-Stark.github.io/master/_posts/pic2.jpg)

קיימות מספר גישות לפתור השאלה הזאת. בין היתר, ברוטפורס בזבזני....
כמובן שניתן לשכלל פתרון כזה ולנסות לצמצם חלק מהאופציות בדרך זו או אחרת. דרך נוספת היא שימוש באלגוריתם בשם Kadane, על שם ממציאו - Jay Kadane. בניגוד לאלגוריתמים 
הקודמים שציינתי, אלגוריתם זה פועל מהר למדי - O(n).

**עדכון:** 
אכתוב גרסה נכונה ועובדת שמחזירה עבור המערך הנתון בשאלה את התוצאה הנכונה - 18. אכתוב כרגע בפייתון, וללא הסבר.
**עדכון:** אילו רק הייתי בודק, הייתי מגלה שלא היה צורך במימוש כזה בפייתון, מכיוון שגם המימוש המקורי מחזיר 18 בדיוק.

<div dir="ltr">
{% highlight python %}
arr = [2, 3, -4, 7, 9, 1]
curr_sum = 0
max_sum = -2**32
for i in range(len(arr)):
    curr_sum = max(arr[i], arr[i]+curr_sum)
    if curr_sum > max_sum:
        max_sum = curr_sum
print(max_sum) # evaluated to 18

{% endhighlight%}
</div>


כאו מופיע תיאור שגוי של האלגוריתם:
<strike>
באלגוריתם הזה מסתכלים רק על רצפים חיוביים במערך, ומחזיקים את הרצף המקסימלי שהתקבל עד עכשיו.
**אבל צריך להבין למה אני מתכוון כשאני אומר רצף חיובי**. אני מתכוון רצף שהסכום שלו חיובי, לא רצף שכל המספרים בו חיוביים.
 אם הסכום שהתקבל עד עכשיו לא חיובי, ויש מספרים חיוביים במערך, ברור שזהו לא פתרון אופטימאלי, ושמכאן גם לא יתחיל פתרון אופטימאלי, ואפשר להתחיל לחשב את הסכום של תת המערך הבא.
 למה זה כל-כך ברור? נניח שהסכום המקסימלי כן היה מתקבל בזכות הערכים הבאים במערך. אם הם כל-כך גדולים, אז המספרים שבאו קודם, שיצרו סכום שלילי, רק מפריעים להם, רק יכולים להוריד את הסכום הכולל. זה לא אומר שאי אפשר שיהיו מספרים שליליים באמצע, זה לא אומר שאי אפשר שהסכום יירד ואחר כך יפוצה ע"י המספרים הבאים, אבל אם *הסכום* הופך מחיובי לשלילי זה אומר שהרצף נקטע, והמערך יכול להיות מחולק לשני מועמדים (או יותר) אפשריים. או החלק לפני המספר שהפך את הרצף לשלילי, או החלק אחרי המספר שהפך את הרצף השלילי.
 אם באמת נתסכל על החלוקה הזאת, זה יהיה ברור. נסתכל על הרצף שכולל את שני החלקים. אם היו שני רצפים עם ערך חיובי כל אחד, ומחברים אותם לרצף אחד גדול, אפילו אם רצף מסויים קטן יותר, הסכום הכולל עולה. אבל אם מוסיפים רצף שיש לו ערך שלילי, הסכום הכולל יורד!
  (על המקרה בו כל איברי המערך שליליים - עוד מעט)
במילים אחרות מה שהאלגוריתם עושה זה דבר כזה:

1.   תחזיק את הסכום של הרצף שאתה מחשב עכשיו.

2.   תחזיק את הסכום המקסימלי שהתקבל עד עכשיו

3.   עבור על המערך:

        3.1    אם סכום הרצף הנוכחי ועוד האיבר הנוכחי במערך שלילי, אפס את סכום הרצף, רצף "יותר" מקסימלי ממה שקיבלנו עד עכשיו כבר לא יתקבל

        3.2    עדכן את הסכום המקסימלי

4.    החזר את הסכום המקסימלי
{: style="color:DarkSlateGrey; size:110%;"}

יפה, ומה קורה אם כל האיברים שליליים? קודם כל - ניתן בקלות לבדוק מצב כזה - עוברים על המערך. ומה באמת תת המערך המקסימלי במקרה כזה? טוב, ברור שתת המערך המקסימלי הוא האיבר המקסימלי במערך. ואז הסכום המקסימלי הוא ערכו של האיבר המקסימלי, וזה מה שצריך להחזיר. לא באמת צריך את הבדיקה הזאת, הלוגיקה הרגילה מתאימה גם למערך שלילי,
אבל הוספתי את הבדיקה הזאת לשם פשטות ולהיות על הצד בטוח.
הנה דוגמה לאיך כל הסיפור הזה יכול להתממש ב- C :
<div dir="ltr">
{% highlight c %}
#include <stdio.h>
#include <stdlib.h>

int max(int a, int b){
	return (a > b) ? a : b;
}
// finds the sum of the maximal subarray
int Max_Sub_Array(int arr[],int n){
    int curr=0; // holds current subarray sum
    int tot=0;  // holds maximub subarray sum so far
    int positive=0; // to check if we have an all-negative array
    for (int i=0;i<n;++i){
        if (arr[i]>0) 
            positive=1;
        curr=max(0,curr+arr[i]);
        tot=max(curr,tot);
    }
    if(positive==0){
        tot=arr[0];
        for(int i=0;i<n;++i)
            tot=max(tot,arr[i]);
    }
    
   return tot;
}

int main(){
	int arr[] = {2, 3, -4, 7, 9, 1};
	printf("%d\n", Max_Sub_Array(arr, 6));
	return 0;
}


// Printed:
// > 18
{% endhighlight %}

int Max_Comb_Sum(int arr[],int n){
    int sum=0;
    for (int i=0; i<n;++i){
        if(arr[i]>0)
            sum+=arr[i];
    }
    if(sum==0){
        sum=arr[0];
        for(int i=0;i<n;++i)
            sum=max(sum,arr[i]);
    }
    return sum;
}
